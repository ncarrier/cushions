Detailed Description

Main API, for clients willing to add URL-like support to their program.

Three solutions are available:

  • A client can call directly cushions_fopen().
  • He/she can left his program untouched and link it with -Wl,–wrap=fopen.
  • He can call the original fopen() function and launch the program with the environment variable LD_PRELOAD set to libcushions.so.

CUSHIONS_API FILE* cushions_fopen ( const char *  path,
const char *  mode 
)

This function opens the file whose name is the string pointed to by path and associates a stream with it.

Parameters
pathName of the file to open, if it starts with SCHEME://, with SCHEME being an URL scheme for which a scheme handler is registered, then this handler will take care of the opening, otherwise, fopen will be used. Multiple schemes can be chained provided the handler for the outermost schemes do rely themselves on cushions_fopen.
modeOpen mode string, such as "rbe", as described in fopen(3)
Returns
Returns a FILE pointer upon successful completion. Otherwise, NULL is returned and errno is set to indicate the error.
